# K8s Architect — 交互式教学网页设计文档

## 概述

一个面向 K8s 日常使用者的交互式教学网页，通过游戏化模拟让用户理解 Kubernetes 内部工作原理。用户扮演 K8s 控制面，手动完成调度决策、网络配置、存储管理等任务。

## 目标用户

K8s 日常使用者 — 会用 kubectl、部署过应用，但想深入理解 Pod 创建背后发生了什么、网络是怎么通的、存储是怎么挂载的。

## 技术方案

**Canvas 渲染 + 游戏化交互**（方案 C）

- 纯 HTML/CSS/JS，无框架依赖
- Canvas 渲染架构图、动画、交互元素
- D3.js 辅助数据可视化和部分动画
- 部署到 GitHub Pages，零后端
- LocalStorage 持久化用户进度

## 视觉风格

深色科技风（暗色主题），参考 GitHub Dark 配色：
- 背景 `#0d1117`，卡片 `#161b22`，边框 `#30363d`
- 主色 `#58a6ff`（蓝），成功 `#3fb950`（绿），警告 `#d29922`（黄），错误 `#f85149`（红），紫色 `#d2a8ff`，橙色 `#f0883e`
- 全中文 UI 文案

## 游戏化机制

### 等级系统
| 等级 | 名称 | 所需经验 |
|------|------|---------|
| Lv.1 | 学徒 | 0 |
| Lv.2 | 运维工程师 | 500 |
| Lv.3 | 架构师 | 1500 |
| Lv.4 | 大师 | 3000 |

### 经验获取
- 完成挑战：+100 经验
- 首次通关（未使用提示）：+50 经验
- 探索隐藏知识点：+30 经验
- 阅读完整说明：+20 经验
- 做出错误决策：不扣分

### 挑战流程（核心循环）
阅读场景说明 → 交互模拟做决策 → 即时反馈+原理解释 → 经验奖励+解锁

### 模块解锁
按顺序解锁，完成当前模块 60% 以上挑战即可解锁下一模块。

## 五大模块设计

### 模块一：整体架构总览（4 挑战）

让用户理解 Master-Node 架构、各组件角色和通信方式。

| # | 挑战 | 类型 | 描述 |
|---|------|------|------|
| 1 | 认识组件 | 探索型 | 点击每个组件查看功能说明 |
| 2 | 拼装控制面 | 拖拽型 | 拖拽 kube-apiserver、etcd、scheduler、controller-manager 到正确位置 |
| 3 | 组件通信 | 序列型 | 按正确顺序点击组件，模拟请求链路 |
| 4 | Worker 节点 | 拖拽型 | 组装 kubelet + kube-proxy + 容器运行时 |

### 模块二：Pod 创建流程（5 挑战）

从 kubectl run 到容器运行的全链路，用户在关键节点做决策。

| # | 挑战 | 类型 | 描述 |
|---|------|------|------|
| 1 | 发起请求 | 序列型 | 模拟 kubectl 发送 YAML 到 API Server |
| 2 | 认证鉴权 | 选择题 | 选择正确的认证方式通过请求 |
| 3 | 持久化 | 序列型 | 将 Pod 写入 etcd 的过程 |
| 4 | 调度决策 | 决策型 | 根据资源条件选择最优节点 |
| 5 | 启动容器 | 序列型 | kubelet 拉取镜像、创建容器的全流程 |

Canvas 渲染：步骤进度条动画，当前步骤高亮+脉冲，用户点击"下一步"推进流程，在关键节点弹出决策选项。

### 模块三：网络架构（5 挑战）

可视化数据包在 Pod 间、节点间、Service 间的完整路径。

| # | 挑战 | 类型 | 描述 |
|---|------|------|------|
| 1 | 同节点通信 | 动画型 | 理解 veth pair + 网桥 |
| 2 | 跨节点通信 | 动画型 | 数据包经过 VXLAN 隧道 |
| 3 | Service 发现 | 序列型 | 从 DNS 到 iptables 的完整链路 |
| 4 | Ingress 路由 | 配置型 | 配置 Ingress 规则将流量导入正确 Service |
| 5 | 网络策略 | 配置型 | 设置 NetworkPolicy 限制 Pod 间通信 |

Canvas 渲染：数据包沿路径流动动画，每经过一跳（veth → cbr0 → iptables → kube-proxy → Service → 目标 Pod）显示该层作用。

### 模块四：存储架构（4 挑战）

PVC 在存储系统中的"寻路"过程。

| # | 挑战 | 类型 | 描述 |
|---|------|------|------|
| 1 | 存储类型 | 选择分类型 | 区分 emptyDir / hostPath / PVC |
| 2 | 动态供应 | 配置型 | 配置 StorageClass 创建 PV |
| 3 | 绑定挂载 | 拖拽型 | PVC 绑定 PV 并挂载到 Pod |
| 4 | 数据生命周期 | 决策型 | 模拟 Pod 删除后数据的不同命运 |

Canvas 渲染：存储层级树形图（CSI → StorageClass → PV → PVC），用户 PVC 在层级中向下寻路。右侧显示 PVC 生命周期进度。

### 模块五：安全与可观测性（4 挑战）

API 请求经过安全链路的完整过程。

| # | 挑战 | 类型 | 描述 |
|---|------|------|------|
| 1 | 认证链路 | 决策型 | 选择正确的认证方式通过 API 请求 |
| 2 | RBAC 权限 | 配置型 | 配置 Role + RoleBinding 授予最小权限 |
| 3 | 安全上下文 | 配置型 | 为 Pod 设置 SecurityContext 限制 |
| 4 | 可观测性三支柱 | 序列型 | 部署监控 + 日志 + 链路追踪 |

Canvas 渲染：请求流经安全关卡（认证 → RBAC → 准入控制 → 执行），每关弹出决策选项。可观测性部分展示 Metrics / Logs / Traces 三支柱的协作关系。

## 交互类型定义

| 类型 | 机制 | 适用场景 |
|------|------|---------|
| 探索型 | 点击组件查看详情，无对错 | 认识架构组件 |
| 拖拽型 | 拖拽元素到目标位置 | 组装架构、建立关系 |
| 序列型 | 按正确顺序点击步骤 | 理解流程、请求链路 |
| 决策型 | 根据条件选择最优方案 | 调度、策略决策 |
| 动画型 | 观察+理解，点击控制播放 | 数据包流动、组件通信 |
| 配置型 | 填写/选择配置参数 | YAML 配置、网络策略 |

## 非功能性需求

- **零后端**：纯静态部署，LocalStorage 存储进度
- **响应式**：适配桌面端（主要），平板可用
- **性能**：Canvas 60fps 动画，流畅拖拽
- **可访问性**：Canvas 外层提供文字替代说明
- **离线可用**：Service Worker 缓存所有资源
